        - action : 제출할 경로

        2) input 태그 : 값을 입력하는 양식 제공
        - name : 서버로 제출시 key 역할
        - 입력된 값 또는 value : 서버로 제출 시 value 역할

GET 방식 요청 및 응답

        GET 방식 요청 
        - 요청하는 주소 뒤에 전달하려는 값을 key=value 형태로 붙여
        서버로 전달하는 방식

        http://localhost/example1.do?inputName=홍길동99&inputAge=25

        http:// -> 프로토콜 (규약)

        localhost/example1.do -> 요청 주소

        ?inputName=홍길동99&inputAge=25 -> queryString(쿼리스트링)

        
        [장점]
        - 단순함
        - 주소에 값을 임의로 작성 가능
        - 캐싱(caching, 저장) 가능
                -> 주소 저장 == 북마크, 즐겨찾기, 주소 공유


        [단점]
        - 주소창 글자 수 제한이 있다
        - 보안에 취약하다(비밀번호가 주소에 보임)
